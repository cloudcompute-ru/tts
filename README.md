# Синтез речи и клонирование голоса (TTS)

Готовый launcher для синтеза речи с клонированием голоса на NVIDIA-видеокарте. Поднимает веб-интерфейс на [Gradio](https://www.gradio.app/) на `0.0.0.0:7860` одной командой — без ручной установки моделей и зависимостей.

Поддерживает три движка на выбор (переключается переменной `CC_APP_MODEL`):

| Движок | Языки | Лицензия модели | Заметки |
|---|---|---|---|
| **XTTS-v2** (по умолчанию) | 17 языков, включая русский, английский, арабский | Coqui Public Model License (CPML) | Зеро-шот клонирование по образцу, универсальный выбор |
| **F5-TTS** | английский, китайский | CC-BY-NC | Высокая естественность; некоммерческая лицензия — для личного использования на арендованной видеокарте |
| **Chatterbox** | преимущественно английский | MIT (Resemble AI) | Лёгкая модель с клонированием голоса |

## Что нужно

Linux с одной NVIDIA GPU от **8 ГБ VRAM** (RTX 3060, RTX 4070 Ti, RTX 3090, RTX 4090, A5000 — все подходят) и актуальный CUDA-драйвер. Синтез речи нетребователен к памяти; 8 ГБ хватает для real-time генерации любым из движков.

## Запуск

```bash
git clone https://github.com/cloudcompute-ru/tts.git
cd tts
# Движок по выбору: xtts-v2 (по умолчанию) | f5-tts | chatterbox
CC_APP_MODEL=xtts-v2 bash provision.sh
```

После сообщения `provisioning complete` откройте `http://<host>:7860/` в браузере. Загрузите чистый образец голоса 10–15 секунд (один говорящий, без музыки и шума), впишите текст и нажмите **Синтезировать**.

`provision.sh` идемпотентен: если веса модели уже скачаны в HF-кэш, повторный запуск пропускает загрузку.

## Качество клонирования

Результат напрямую зависит от качества образца. Используйте чистую запись **10–15 секунд**: один голос, без музыки, без фонового шума, без реверберации. Слишком короткий или зашумлённый образец заметно ухудшает естественность и схожесть.

## Что внутри

- `provision.sh` — ставит выбранный движок TTS + Gradio, скачивает веса в HF-кэш, поднимает веб-интерфейс на `0.0.0.0:7860`. Для XTTS-v2 и Chatterbox встроенный Gradio-интерфейс генерируется скриптом; F5-TTS использует собственный интерфейс (`f5-tts_infer-gradio`).

## Переменные окружения

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `CC_APP_MODEL` | `xtts-v2` | Движок: `xtts-v2`, `f5-tts` или `chatterbox` |
| `TTS_PORT` | `7860` | Порт веб-интерфейса |
| `CC_PROVISION_URL` | — | Эндпоинт для отчётов о стадиях (выставляет панель cloudcompute.ru) |
| `CC_AGENT_TOKEN` | — | Bearer-токен для этого эндпоинта |

При запуске вручную (без `CC_PROVISION_URL` / `CC_AGENT_TOKEN`) отчёты о стадиях молча пропускаются — скрипт работает автономно.
